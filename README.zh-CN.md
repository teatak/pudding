# Pudding

Pudding 是一个 local-first 的桌面 AI 工作区，适合处理不止停留在聊天窗口里的任务。你可以一边与模型对话，
一边把有价值的结果留在共享画布中，并自然地使用浏览器、小组件、技能和语音能力。

[English](./README.md) · [下载最新版本](https://github.com/teatak/pudding/releases/latest)

> Pudding 目前处于早期预览阶段。这个公开仓库用于提供产品说明、发布包和初始目录，源码暂未公开。

## 主要能力

- **多会话工作区**：同时保留多个独立会话及其项目上下文。
- **共享画布**：持续保留文档、表格、图片、小组件和浏览器页面等结果。
- **可见的工具过程**：在画布中使用浏览器、终端、相机、桌面截取和已安装应用。
- **可扩展工作流**：通过小组件、技能、MCP 工具和应用集成扩展特定任务。
- **灵活的模型接入**：支持 OpenAI-compatible API、OpenRouter、DeepSeek、Gemini、Ollama 等模型服务。
- **可选语音能力**：安装对应运行资源后，可使用听写、语音对话和原音播放。

## 安装

当前预览版本支持搭载 Apple 芯片的 macOS。

1. 从 [Releases](https://github.com/teatak/pudding/releases/latest) 下载
   `Pudding-<version>-arm64.dmg`。
2. 打开 DMG，将 `Pudding.app` 拖入“应用程序”。
3. 打开 Pudding。

### macOS 首次打开

当前预览包使用 ad-hoc 签名，尚未经过 Apple 公证。如果 macOS 显示“Apple 无法验证 Pudding”并阻止打开：

1. 尝试打开 Pudding 一次，然后关闭系统提示。
2. 打开“系统设置 > 隐私与安全性”。
3. 在“安全性”中找到 Pudding，点击“仍要打开”。
4. 验证身份后确认“打开”。

“仍要打开”通常只在被拦截后约一小时内显示。仅对从本仓库 Release 下载的构建执行此操作。具体步骤可参考
[Apple 官方说明](https://support.apple.com/guide/mac-help/open-a-mac-app-from-an-unknown-developer-mh40616/mac/26)。

## 本地数据

Pudding 将应用状态、配置、数据库和运行资源保存在本机：

```text
~/.pudding   # 应用数据、配置、数据库、运行资源和缓存
~/pudding    # 默认用户工作目录
```

模型请求会发送到你选择的模型服务。模型凭据和 provider 配置均在 Pudding 内管理。

## 语音运行资源

语音功能使用保存在 `~/.pudding/runtime` 下的可选运行资源。Pudding 会在安装前提示，因此桌面安装包无需默认
携带体积较大的语音模型。语音资源通过独立的 `runtime-v1` Release 发布。

## 小组件与技能

- **小组件**是可复用的画布体验，可以从小组件库安装、从包导入，或从画布内容保存。
- **技能**提供可跨会话复用的任务说明和工作流程。
- **预览内容**在对话中保持临时状态，只有在你主动保留或安装后才会成为长期内容。

## 发布与更新

- 桌面应用使用 `v0.1.1` 这类版本标签。
- 运行资源使用 `runtime-v1` 这类版本标签。
- 未签名的预览版本默认采用手动更新：Pudding 可以提示发现新版本并打开 Release 页面，但不会静默替换当前应用。

桌面应用和运行资源分开发布，避免每次应用更新都重复下载较大的语音资源。历史版本会继续保留，方便必要时回退。

会随着预览版本推进逐步补充。
